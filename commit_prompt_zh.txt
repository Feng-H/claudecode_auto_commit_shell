## 输出要求

### 1. 格式规范（不要用代码块包裹）
type(scope): subject

icon 详细描述1
icon 详细描述2

### 2. Type 类型（仅限以下8种）
feat ✨ - 新功能
fix 🐛 - Bug修复
docs 📝 - 文档更新
style 💄 - 代码格式调整（不影响功能）
refactor 🔧 - 重构（不改变功能）
perf ⚡ - 性能优化
test ✅ - 测试相关
chore 🧹 - 构建/工具/配置

### 3. 内容要求
- Subject: 不超过50字符，使用动词原形（添加/修复/更新/优化等）
- Scope: 模块名（从项目上下文推断，如 core/ui/api/auth）
- Body: 每点不超过30字符，说明做什么+为什么
- 价值导向: 强调业务价值，而非代码细节
- 技术准确: 根据实际变更准确判断类型

### 4. 类型判断准则
- 新增文件/功能 → feat
- 修复报错/逻辑 → fix
- 修改文档 → docs
- 空格/缩进/格式 → style
- 结构调整/重命名 → refactor
- 速度/缓存优化 → perf
- 测试代码 → test
- 配置/依赖 → chore

### 5. 输出示例

示例1 - 新功能:
feat(auth): 实现JWT用户认证
✨ 新增token生成和验证机制
✨ 支持用户登录/注册接口
🔧 密码使用bcrypt加密存储

示例2 - Bug修复:
fix(api): 修复查询条件错误
🐛 修正WHERE语句逻辑错误
🐛 处理空值边界情况
✅ 补充单元测试覆盖

示例3 - 文档:
docs(readme): 更新部署文档
📝 补充Docker部署步骤
📝 添加环境变量配置说明
📝 修正示例代码格式

示例4 - 重构:
refactor(user): 重构用户数据结构
🔧 统一用户信息字段定义
🔧 简化数据访问层逻辑
✅ 更新相关测试用例
